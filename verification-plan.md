# 従業員管理ページ 検証プラン

ラクミーウェブ経営管理（会社スコープ）従業員管理機能のデザインモック（`feature/eemgt`）に対する検証プラン。
仕様の正本は `employee_management_system_specification_designmock.md`。本プランは**画面別テストケース**と、各ケースの実行に必要な**前提入力データ（`data/`）**で構成する。

> 注記: 本モックは静的HTMLのため、DB保存/サーバ計算は発生しない。検証は **UI挙動・表示・遷移・バリデーション表現・JS切替** の確認に限定する。サーバ実装後は同じケースをE2E（実データ投入）へ昇格させる前提。

### データ設計の要点
- **雇用区分**：正社員 / パート / アルバイト を最低1名ずつ（バッジ色分け確認）。
- **給与単位**：月給（200,000–500,000）と時給（1,000–1,500）を混在（バッジ `badge-dark`/`badge-light-gray`、通貨ソート確認）。
- **交通費単位**：日額（500–2,000）／月額（10,000–30,000）混在。
- **ヘルプ先店舗あり/なし**を混在（あり＝EMP-001/003/005/008/009/011）→ ヘルプ先交通費の動的フォーム表示有無を検証。
- **設定変更履歴**に「適用済」「予約中」を含める → ステータス表示と「更新予約」動作の対象。
- **変更ソース**に 手動編集 / 更新予約 / CSVインポート(KingOfTime) / CSVインポート(ジョブカン) を含める → フィルタ検証。
